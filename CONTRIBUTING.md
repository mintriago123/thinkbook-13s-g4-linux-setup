## How to Contribute

### Reporting Issues
- Check if the issue already exists
- Provide detailed information about your system
- Include error messages and logs when relevant
- Specify which Linux distribution you're using

### Suggesting Enhancements
- Clearly describe the enhancement
- Explain why it would be useful
- Provide examples if possible

### Submitting Pull Requests

1. **Fork the repository**
   ```bash
   git clone https://github.com/mintriago123/thinkbook-13s-g4-linux-setup.git
   ```

2. **Create a new branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Keep changes focused and atomic
   - Test your changes thoroughly
   - Update documentation if needed

4. **Commit your changes**
   ```bash
   git commit -m "Add: brief description of your changes"
   ```
   
   Use conventional commit messages:
   - `Add:` for new features
   - `Fix:` for bug fixes
   - `Update:` for updates to existing features
   - `Docs:` for documentation changes

5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Open a Pull Request**
   - Provide a clear description of your changes
   - Reference any related issues
   - Wait for review and address feedback

## Guidelines

### Documentation
- Write clear, concise documentation
- Include both English and Spanish versions when possible
- Use code blocks for commands and examples
- Test all commands before submitting

### Code Style
- Use 4 spaces for indentation in shell scripts
- Add comments for complex operations
- Keep scripts POSIX-compliant when possible

### Testing
- Test on multiple distributions if possible
- Verify commands don't have unintended side effects
- Document any prerequisites or dependencies

## Areas Where We Need Help

- Testing on different Linux distributions
- Translations to other languages
- Additional hardware configurations
- Performance optimization tips
- Troubleshooting guides

## Questions?

Feel free to open an issue for:
- Clarification on contribution guidelines
- Questions about the project
- Discussion about potential features
